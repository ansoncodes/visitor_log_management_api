# Visitor Log Management API

A backend API built using Django and Django REST Framework to manage visitors entering an office.

---

## Features

- Visitor check-in
- Visitor check-out
- List visitors currently inside
- List visitors by date

---

## Tech Stack

- Python
- Django
- Django REST Framework
- SQLite
---

## Setup Instructions

### 1. Clone the repository

```bash
git clone https://github.com/ansoncodes/visitor_log_management_api.git
cd visitor_log_management_api
```

### 2. Create and activate a virtual environment

```bash
python -m venv venv
venv\Scripts\activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Run migrations

```bash
python manage.py migrate
```

### 5. Start the development server

```bash
python manage.py runserver
```

---

## API Endpoints

### 1. Check-in a Visitor

**POST** `/api/visitors/check-in/`

**Request Body**

```json
{
  "name": "Anson Antony",
  "phone": "1234567890",
  "purpose": "Interview"
}
```

---

### 2. Check-out a Visitor

**POST** `/api/visitors/check-out/`

**Request Body**

```json
{
  "phone": "1234567890"
}
```

---

### 3. List Visitors Currently Inside

**GET** `/api/visitors/inside/`

**Response**

```json
[
  {
    "id": 1,
    "name": "Anson Antony",
    "phone": "1234567890",
    "purpose": "Interview",
    "check_in_time": "2026-02-09T10:30:00Z",
    "check_out_time": null
  }
]
```

---

### 4. List Visitors by Date

**GET** `/api/visitors/by-date/?date=YYYY-MM-DD`

**Example**

```
/api/visitors/by-date/?date=2026-02-09
```

---

## Business Rules

- A visitor cannot check in again without checking out
- Phone number must be unique per active visit
- Check-in time is auto-generated by the system
- Check-out time is set explicitly during checkout
